# 核心工程信条

## 1. 文档优先

设计、计划、接口契约必须先于大规模编码。

## 2. 简单优先

优先选择“清晰、稳定、易维护”的方案，而不是“看起来先进”的方案。

## 3. 模块化单体优先

在证据不足前，不提前拆微服务。

## 4. 业务真相只保留一份

业务规则、权限裁决、状态流转统一保留在后端。

## 5. 契约先行

接口、错误码、事件模型统一沉淀到共享契约中。

## 6. 外部依赖必须隔离

微信、短信、校园系统、对象存储全部走 provider / connector 层。

## 7. 可观测性必须内建

日志、指标、追踪、审计不是上线后补的功能。

## 8. 面向 AI 的仓库组织

目录、命名、计划、设计文档必须让 AI 能快速建立准确上下文。

## 9. 文档要持续修剪

过期文档比没有文档更危险，必须维护新鲜度。

## 10. 质量要可量化

通过质量评分、测试覆盖、接口稳定性、变更记录持续衡量工程状态。
