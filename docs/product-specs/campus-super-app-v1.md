# 校园综合应用 V1 产品范围

## 1. 产品目标

建设一个覆盖学生、教师、运营人员、管理员的校园综合应用，提供统一入口、统一身份、统一消息和统一运营后台。

## 2. 用户角色

- 学生
- 教师
- 学院管理员
- 校级管理员
- 运营编辑

## 3. V1 核心场景

- 微信或手机号登录并绑定校园身份
- 查看首页公告、轮播、校园资讯
- 查看课程表、考试安排、成绩
- 查看活动并完成报名
- 提交报修或办事申请
- 接收站内消息和通知
- 管理员发布内容、管理活动、处理工单、查看统计

## 4. MVP 必做范围

- 统一认证与身份绑定
- 用户、角色、权限
- 门户内容与公告管理
- 课程表基础接入
- 活动管理与报名
- 报修 / 工单基础流程
- 消息中心
- 管理后台基础数据看板

## 5. 暂不纳入 V1

- 复杂审批引擎
- 电商化商城
- 即时聊天系统
- 大规模多租户 SaaS 化能力

## 6. 成功指标

| 指标 | 目标 |
| --- | --- |
| 首月活跃用户渗透率 | `> 40%` |
| 登录成功率 | `> 99.5%` |
| 课程信息查询成功率 | `> 99%` |
| 活动报名成功率 | `> 99.5%` |
| 管理后台内容发布成功率 | `> 99%` |

## 7. 核心验收标准

- 同一用户在小程序和 Flutter App 中看到一致的核心业务状态
- 管理员角色切换后权限立即生效
- 课程和活动数据在约定同步窗口内可用
- 工单全流程具备审计记录
- 关键页面在网络异常下有明确降级提示
