# 校园综合应用 V1 产品范围

## 1. 产品目标

建设一个覆盖学生、教师、运营人员、管理员的校园综合应用，先围绕校园生活信息撮合与轻交易场景建立统一入口、统一身份、统一消息和统一运营后台，后续再扩展教务能力。

## 2. 用户角色

- 学生
- 教师
- 学院管理员
- 校级管理员
- 运营编辑

## 3. V1 核心场景

- 默认通过微信小程序登录并建立用户会话
- 完成教务绑定后解锁联系方式查看能力
- 浏览与发布跑腿信息
- 浏览与发布组局信息（含拼车等轻社交撮合）
- 浏览与发布二手交易信息
- 浏览与发布资料信息
- 浏览与发布失物招领信息
- 接收站内消息和通知
- 管理员进行内容审核、用户治理和数据统计

## 4. MVP 必做范围

- 微信小程序登录与统一账号体系
- 教务绑定与联系方式访问控制
- 用户、角色、权限
- 门户内容与公告管理
- 跑腿模块
- 组局模块（含拼车等轻社交撮合）
- 二手交易模块
- 资料模块
- 失物招领模块
- 消息中心
- 管理后台基础数据看板与审核能力

## 5. 暂不纳入 V1

- 完整教务能力接入（课程表、考试安排、成绩查询）
- 复杂审批引擎
- 电商化商城
- 即时聊天系统
- 大规模多租户 SaaS 化能力

## 6. 成功指标

| 指标 | 目标 |
| --- | --- |
| 首月活跃用户渗透率 | `> 40%` |
| 登录成功率 | `> 99.5%` |
| 校园生活信息发布成功率 | `> 99%` |
| 联系方式解锁成功率 | `> 99.5%` |
| 管理后台审核操作成功率 | `> 99%` |

## 7. 核心验收标准

- 同一用户在小程序和 Flutter App 中看到一致的核心业务状态与联系方式可见性
- 未绑定教务的账号无法通过任何客户端获取联系方式字段
- 完成教务绑定后，联系方式查看权限在约定时延内生效
- 跑腿、组局、二手交易、资料、失物招领五类信息都具备可发布、可浏览、可审核的基础闭环
- 管理员角色切换后权限立即生效
- 关键页面在网络异常下有明确降级提示
