# portal

当前已落地：

- 门户首页聚合接口：轮播与公告摘要
- 公告列表与公告详情
- 管理端公告发布、更新、删除接口
- 管理端轮播列表、详情、创建、更新、删除接口
- `memory / postgres` 双后端；`postgres` 模式下内置轮播与公告种子数据

约束：

- 门户内容读取不依赖客户端本地规则推断
- 管理发布权限以后端 `portal:publish` 为准
- 当前先保持简单公告模型，后续再扩展资讯、栏目与更复杂编排

当前边界：

- 当前轮播与公告均已具备管理端维护路径
- 暂未实现更复杂的栏目、资讯正文富文本与定时发布
- 暂未实现独立“文章/资讯”模型，管理后台中的文章页当前仅作为预留入口
