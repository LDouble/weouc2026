# notification

当前已落地：

- 当前用户通知列表
- 未读通知数量
- 单条通知已读回执
- 管理端站内通知发布接口
- 当前主链路为 `MongoDB`

约束：

- 通知收件范围由后端 `target_scope` 与 `target_user_ids` 裁决
- 通知发布权限以后端 `notification:publish` 为准
- 已读状态按用户维度记录，不依赖客户端自行合并

当前边界：

- 当前只落地站内通知，不含短信、微信订阅消息和推送通道
- 暂未接入短信、微信订阅消息和推送通道的投递状态持久化

目标重构方向（设计阶段）：

- 通知内容、目标范围、投放配置迁移到 `MongoDB`
- 已读态、热点计数与幂等控制继续允许使用 `Redis` 做短期状态优化
- 后续如果扩展多通道投递，投递日志与回执统一纳入 `MongoDB` 追踪
