# notification

当前已落地：

- 当前用户通知列表
- 未读通知数量
- 单条通知已读回执
- 管理端站内通知发布接口
- `memory / postgres` 双后端；`postgres` 模式下内置通知种子数据

约束：

- 通知收件范围由后端 `target_scope` 与 `target_user_ids` 裁决
- 通知发布权限以后端 `notification:publish` 为准
- 已读状态按用户维度记录，不依赖客户端自行合并

当前边界：

- 当前只落地站内通知，不含短信、微信订阅消息和推送通道
- 暂未接入短信、微信订阅消息和推送通道的投递状态持久化
