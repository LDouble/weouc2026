# campus_life

当前已落地：

- 首页动态流
- 二手：列表、详情、发布、收藏
- 跑腿：列表、详情、发布、接单、取消发布、取消接单
- 资料：列表、详情、发布
- 失物招领：列表、详情、发布
- 拼车：列表、详情、发布
- 五类内容统一 `review_status`
- 审核列表与审核状态更新接口
- `memory / postgres` 双后端切换

约束：

- 业务规则只在 `service` 层
- 数据访问只经 `repo`
- 文件访问只保存稳定对象路径，展示时由后端按需签 URL
- 公开列表与详情只展示 `published` 内容；发布者与审核员可继续访问待审内容

当前边界：

- 更通用的 `meetup/组局` 还未在后端落地
- 管理端审核页面与更完整的审核审计记录还未落地，当前先提供最小审核接口
- 当前 `postgres` 后端默认不会自动插入演示数据，首次启动后列表为空，需通过发布接口产生业务记录
